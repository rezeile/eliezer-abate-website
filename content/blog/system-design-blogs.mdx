##### Classic Papers

Paper
Topic
Authors
What is it?

[Amazon Dynamo](https://www.cs.cornell.edu/courses/cs5414/2017fa/papers/dynamo.pdf)
Distributed Storage
Amazon
A highly available distributed data store built for
Amazon's massive scale.

[Big Table](https://static.googleusercontent.com/media/research.google.com/en//archive/bigtable-osdi06.pdf)
Distributed Storage
Google
A distributed storage system for managing structured
data that is designed to scale to a very large size: petabytes of data
across thousands of commodity servers

[Google File System](https://pdos.csail.mit.edu/6.824/papers/gfs.pdf)
Distributed Storage
Google
A scalable distributed file system for large
distributed data-intensive applications

[Memcache at Facebook](https://www.usenix.org/system/files/conference/nsdi13/nsdi13-final170_update.pdf)
Distributed Storage
Facebook
How Facebook leverages memcached as a building block
to construct and scale a distributed key-value store that supports the
world’s largest social network.

[Cassandra](https://www.cs.cornell.edu/projects/ladis2009/papers/lakshman-ladis2009.pdf)
Distributed Storage
Facebook
A distributed storage system for managing very large
amounts of structured data spread out across many commodity servers,
while providing highly available service with no single point of
failure.

[Spanner](https://pdos.csail.mit.edu/6.824/papers/spanner.pdf)
Distributed Storage
Google
A scalable, multi-version, globallydistributed, and
synchronously-replicated database.

[Map Reduce](https://pdos.csail.mit.edu/6.824/papers/mapreduce.pdf)
Distributed Processing
Google
A programming model and an associated implementation
for processing and generating large data sets

[Spark](https://pdos.csail.mit.edu/6.824/papers/zaharia-spark.pdf)
Distributed Processing
UC Berkeley
A distributed memory abstraction that lets
programmers perform in-memory computations on large clusters in a
fault-tolerant manner

[Apache Kafka](https://cs.uwaterloo.ca/~ssalihog/courses/papers/netdb11-final12.pdf)
Distributed Messaging
LinkedIn
A distributed messaging system for collecting and
delivering high volumes of log data with low latency

[Raft](https://pdos.csail.mit.edu/6.824/papers/raft-extended.pdf)
Distributed Consensus
Stanford
A consensus algorithm for managing a replicated log.
[See MIT Lecture for more.](https://www.youtube.com/watch?v=64Zp3tzNbpE&amp;t=88s)

[TAO](https://www.usenix.org/system/files/conference/atc13/atc13-bronson.pdf)
Architecture
Facebook
A simple data model and API tailored for serving
Facebook's massive social graph, and TAO, its implementation.

[Zookeeper](https://pdos.csail.mit.edu/6.824/papers/zookeeper.pdf)
Architecture
Yahoo
A service for coordinating processes of distributed
applications.

[Google Search](http://infolab.stanford.edu/~backrub/google.html)
Architecture
Stanford
Sergey and Larry Page's The Anatomy of a
Hypertextual Web Search Engine Describing Google's Early
Architecture.

[Bitcoin](https://pdos.csail.mit.edu/6.824/papers/bitcoin.pdf)
Architecture
Satoshi Nakomoto
The White Paper that introduced Bitcoin to the
world.

[CAP Theorem](https://www.infoq.com/articles/cap-twelve-years-later-how-the-rules-have-changed/)
Reflections
Eric Brewer
Eric Brewer Reflecting on the CAP Theorem.
